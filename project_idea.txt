  How The App Would Work

  1. User creates a ticket through the API Gateway.
  2. Gateway sends request to Ticket Service.
  3. Ticket Service saves ticket in PostgreSQL.
  4. Ticket Service publishes TicketCreated event to Kafka.
  5. Assignment Service consumes the event and assigns the ticket.
  6. Assignment Service publishes TicketAssigned.
  7. Notification Service consumes events and sends updates.
  8. Audit Service records all lifecycle events.
  9. Jenkins builds, tests, and deploys every committed change.


NOW:

Controller = talks HTTP
Service = performs use cases
Domain = protects business rules
Repository/database = stores data, later
Kafka = sends events to other services, later
